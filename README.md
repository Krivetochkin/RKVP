# Святое Торжище
Ссылка: https://svyattorg.by/

**Святое Торжище** — это специализированный интернет-магазин, предлагающий широкий ассортимент православной продукции для верующих и интересующихся культурой. Платформа предназначена для удобного поиска и приобретения икон, духовной литературы, церковной утвари и подарков с религиозным смыслом. Мы стремимся сделать православные товары доступными для покупателей по всей стране, обеспечивая качественный сервис и достоверность информации о каждом изделии. Сайт также служит просветительским ресурсом, помогая пользователям правильно ориентироваться в выборе церковных принадлежностей.

## Целевая аудитория

*   Практикующие православные христианы, желающие приобрести товары для личной молитвы и церковного быта.
*   Люди, подбирающие тематические подарки к религиозным праздникам (Пасха, Рождество, Крещение, именины).
*   Интересующиеся православной культурой, историей церкви и духовной литературой, включая студентов и исследователей.

## Ключевой функционал

*   **Каталог товаров:** Удобная навигация по категориям (иконы, книги, свечи, одежда, утварь) с фильтрами по цене, размеру, материалу и назначению.
*   **Корзина и оформление заказа:** Безопасная оплата онлайн, выбор способа доставки и отслеживание статуса заказа в личном кабинете.
*   **Личный профиль пользователя:** История покупок, сохранение избранных товаров в «Отложенное» и управление адресами доставки.
*   **Информационные карточки товаров:** Подробное описание с учетом церковных традиций, рекомендации по выбору иконописных стилей и указание размеров для церковной одежды.
*   **Кнопка «Поставить свечу онлайн»:** Пользователь сможет зажечь виртуальную свечу.
*   **Генератор имен для крещения:** Если родители не могут определиться, то генератор поможет выбрать имя святого из календаря на текущую дату.
*   **Уникальное звуковое и визуальное сопровождение:** В качестве звукового сопровождения будет возможность прослушать православные музыкальные произведени, пока пользователь выбирает товар. В качестве звука уведомления будет звон колоколов.
*   **Случайная цитата:** Пока пользователь оформляет заказ, внизу отображается рандомная мудрая мысль или стих из Псалтыри. Поднимает настроение и настраивает на нужный лад.
*   **Прямая трансляция службы:** Прямо на сайте можно будет просматривать прямую трансляцию службы, если таковая проходит сейчас.
*   **Фильтр товаров по узким параметрам:** Фильтр скрывает на сайте товары, содержащие кожу (например, дорогие переплеты из натуральной кожи), если человек выбрал категорию "пост".

## 3.3. Технологический стек

### Frontend

| Технология | Обоснование выбора |
|-----------|-------------------|
| **Фреймворк: Vue 3 (Composition API)** | Обеспечивает высокую производительность и реактивность интерфейса. Composition API позволяет удобно переиспользовать логику (например, логику фильтрации товаров в «постном режиме» или управления виртуальной свечой), что критично для сложных форм оформления заказа. Vue 3 имеет мягкую кривую обучения и отличную документацию на русском языке. |
| **State-менеджмент: Pinia** | Современный стандарт для Vue 3, пришедший на смену Vuex. Обеспечивает строгую типизацию (TypeScript-friendly) и удобное управление глобальным состоянием: хранение данных авторизованного пользователя, содержимого корзины, списка избранных товаров, настроек «постного режима» и звуковых предпочтений. |
| **Роутинг: Vue Router** | Необходим для создания Single Page Application (SPA). Обеспечивает плавный переход между разделами без перезагрузки страницы, позволяет легко настроить защиту приватных маршрутов через navigation guards (например, редирект неавторизованных пользователей с `/profile` на `/auth/login`). Поддерживает динамические маршруты для карточек товаров (`/catalog/:id`). |
| **HTTP-клиент: Axios** | Библиотека для работы с API. В отличие от стандартного fetch, позволяет удобно настраивать базовый URL сервера, перехватчики запросов/ответов (для добавления токена авторизации) и централизованно обрабатывать ошибки ответов (например, показывать единый формат сообщений об ошибках). |
| **UI-библиотека: Vuetify 3 / Element Plus** | Готовые компоненты для ускорения разработки: таблицы, формы, модальные окна, карточки товаров, навигационные меню. Адаптивная вёрстка из коробки (mobile-first), что важно для аудитории разного возраста. Тема настраивается под православную цветовую гамму (золото, бордо, кремовый). |

### Backend

| Технология | Обоснование выбора |
|-----------|-------------------|
| **Среда выполнения и фреймворк: Node.js + Express** | Асинхронная неблокирующая архитектура Node.js отлично подходит для I/O операций, таких как запросы к базе данных товаров, обработка заказов и работа с файлами (загрузка изображений икон, книг). Express предоставляет минималистичный и гибкий инструментарий для быстрого построения REST API эндпоинтов с минимальным количеством кода. Единый язык JavaScript/TypeScript на фронтенде и бэкенде упрощает разработку. |
| **Инструментарий: dotenv, nodemon, cors** | **dotenv** обеспечивает безопасное хранение ключей доступа к базе данных и секретных переменных окружения (не попадают в репозиторий). **nodemon** ускоряет разработку за счёт автоматической перезагрузки сервера при изменении кода. **cors** разрешает взаимодействие между фронтендом и бэкендом, размещёнными на разных доменах/портах. |
| **База данных и Backend-as-a-Service: Supabase (PostgreSQL)** | Использование строгой реляционной базы данных (PostgreSQL) идеально подходит для связи сущностей: Пользователь → Заказ → Товар → Категория → Отзыв. Supabase берёт на себя рутину по настройке безопасной аутентификации пользователей (JWT-токены, восстановление пароля), предоставляет удобное хранилище (Storage) для загрузки изображений товаров и икон, а также имеет встроенную реальную подписку (Realtime) для будущих функций (например, уведомления о статусе заказа). |

### 1. Страница: Главная (`/`)

**Тип доступа:** публичный

**Основные блоки:**

| № | Блок | Описание | Источник данных |
|---|------|----------|----------------|
| 1 | Шапка | Логотип, меню (Каталог, Свеча, Трансляция, Поиск), профиль/кнопка «Войти» | `GET /api/user/me` (если авторизован) |
| 2 | Герой-секция | Заголовок, икона дня с кратким описанием, CTA-кнопки «В каталог», «Зажечь свечу» | Статичный контент + `GET /api/icon-of-day` |
| 3 | Виджет «Виртуальная свеча» | Мини-форма: имя, тип (здравие/упокой), кнопка «Зажечь» | `POST /api/candles` |
| 4 | Популярные категории | Карусель из 5-6 категорий с иконками (Иконы, Книги, Свечи, Одежда, Утварь) | `GET /api/categories` |
| 5 | Товары дня / Акции | Сетка из 4-6 карточек товаров со скидкой | `GET /api/products?promoted=true` |
| 6 | Цитата дня | Случайная цитата из Псалтыри или святых отцов | `GET /api/quotes/daily` |
| 7 | Анонс трансляции | Блок с информацией о ближайшей службе, кнопка «Напомнить» | `GET /api/live/schedule` |
| 8 | Футер | Ссылки на разделы, контакты, соцсети, режим «Пост» | Статичный контент |

**Динамический контент и состояния:**

🔹 **Товары дня:**
*   **Загрузка:** скелетоны карточек (6 шт.)
*   **Пусто:** «Сегодня нет акционных товаров» + кнопка «Смотреть каталог»
*   **Ошибка:** «Не удалось загрузить» + кнопка «Повторить»

🔹 **Цитата дня:**
*   **Загрузка:** текст-заглушка с анимацией мерцания
*   **Ошибка:** показать запасную цитату из кэша

🔹 **Виджет свечи:**
*   **Успех:** анимация зажжённой свечи, сообщение «Ваша свеча горит»
*   **Ошибка:** «Не удалось зажечь свечу, попробуйте позже»

**Поведение для разных пользователей:**

*   **Гость:** видит все секции, кнопки действий ведут на `/auth/login`
*   **Авторизованный:** в шапке отображается аватар, виджет свечи предлагает «Сохранить в профиль»

### 2. Страница: Аутентификация (`/auth/*`)

**Тип доступа:** публичный

**Подстраницы:** `/auth/login`, `/auth/register`, `/auth/recover`, `/auth/logout`

#### Регистрация (`/auth/register`)

**Основные блоки:**

| № | Блок | Описание | Источник данных |
|---|------|----------|----------------|
| 1 | Шаг 1: Контакты | Поля: email, пароль (с проверкой сложности), подтверждение пароля | Валидация на клиенте + `POST /api/auth/check-email` |
| 2 | Шаг 2: Профиль | Поля: имя, телефон, адрес (опционально), аватар | `GET /api/auth/check-email` (проверка уникальности) |
| 3 | Шаг 3: Preferences | Опрос: дата именин, интересующие категории товаров | `GET /api/preferences/categories` |
| 4 | Завершение | Кнопка «Создать аккаунт», ссылка «Уже есть аккаунт? Войти» | `POST /api/auth/register` |

**Состояния:**
*   **Валидация полей:** подсветка ошибок в реальном времени
*   **Отправка:** спиннер на кнопке, блокировка формы
*   **Успех:** редирект на `/` с тостом «Добро пожаловать!»
*   **Ошибка:** отображение сообщения под соответствующим полем

#### Вход (`/auth/login`)

**Основные блоки:**

| № | Блок | Описание | Источник данных |
|---|------|----------|----------------|
| 1 | Форма входа | Поля: email/телефон, пароль; чекбокс «Запомнить меня» | `POST /api/auth/login` |
| 2 | Дополнительные ссылки | «Забыли пароль?», «Создать аккаунт» | Статичные ссылки |

**Состояния:**
*   **Неправильные данные:** ошибка «Неверный email или пароль»
*   **Слишком много попыток:** капча / временная блокировка
*   **Успех:** сохранение токена, редирект на предыдущую страницу или `/`

#### Восстановление пароля (`/auth/recover`)

**Основные блоки:**

| № | Блок | Описание | Источник данных |
|---|------|----------|----------------|
| 1 | Ввод почты | Поле email, кнопка «Отправить код» | `POST /api/auth/recover/request` |
| 2 | Ввод кода | 6-значный код из SMS/email, таймер повторной отправки (60 сек) | `POST /api/auth/recover/verify` |
| 3 | Новый пароль | Поля: новый пароль, подтверждение | `POST /api/auth/recover/reset` |

### 3. Страница: Каталог (`/catalog`)

**Тип доступа:** публичный

**Основные блоки:**

| № | Блок | Описание | Источник данных |
|---|------|----------|----------------|
| 1 | Хлебные крошки | Навигация: Главная → Каталог → Категория | Статичный контент |
| 2 | Боковая панель фильтров | Цена, размер, материал, назначение, **режим «Пост»** | `GET /api/filters/options` |
| 3 | Сортировка | По популярности / цене / новизне / рейтингу | Параметр `?sort=...` |
| 4 | Сетка товаров | Карточки товаров (изображение, название, цена, рейтинг) | `GET /api/products?...` |
| 5 | Пагинация | Кнопка «Загрузить ещё» или бесконечный скролл | `?offset=...` |

**Динамика и состояния:**

🔹 **Фильтры:**
*   **Режим «Пост»:** при активации скрываются товары с кожей + пояснение «Согласно уставу, в пост...»
*   **Применение фильтра:** автоматический перезапрос результатов без перезагрузки страницы

🔹 **Результаты:**
*   **Загрузка:** скелетоны сетки (8-12 карточек)
*   **Пусто:** «Ничего не найдено» + рекомендации сбросить фильтры
*   **Ошибка:** «Проблема с подключением» + кнопка повтора

### 4. Страница: Детальная страница товара (`/catalog/:id`)

**Тип доступа:** публичный

**Основные блоки:**

| № | Блок | Описание | Источник данных |
|---|------|----------|----------------|
| 1 | Галерея | Основное изображение + миниатюры (свайп/зум) | Вложенный массив `images[]` в ответе API |
| 2 | Заголовок | Название товара, артикул, наличие, рейтинг | `GET /api/products/:id` |
| 3 | Цена и покупка | Цена, кнопка «В корзину», «В отложенное», «Купить в 1 клик» | `POST /api/cart/add` |
| 4 | Описание | Подробное описание с учётом церковных традиций | Поле `description` |
| 5 | Характеристики | Размер, материал, вес, страна производства, освящение | Поля товара |
| 6 | Рекомендации | «С этим товаром покупают», «Похожие товары» | `GET /api/products/:id/similar` |
| 7 | Отзывы | Список отзывов, форма добавления (для авторизованных) | `GET /api/products/:id/reviews`, `POST /api/products/:id/reviews` |
| 8 | Виджет свечи | «Зажечь свечу за этот товар» (доп. опция при покупке) | `POST /api/candles` |

**Динамика и состояния:**

🔹 **Добавление в корзину:**
*   **Неавторизованный:** клик → модалка «Войдите, чтобы сохранить в избранное»
*   **Авторизованный:** анимация добавления, счётчик корзины обновляется

🔹 **Состояния страницы:**
*   **Загрузка:** скелетон галереи + контента
*   **Не найдено:** 404-страница с предложением «Посмотреть другие товары»
*   **Ошибка сети:** «Не удалось загрузить товар» + кнопка повтора

### 5. Страница: Корзина и оформление заказа (`/cart` → `/checkout`)

**Тип доступа:** смешанный

**Основные блоки:**

| № | Блок | Описание | Источник данных |
|---|------|----------|----------------|
| 1 | Список товаров | Карточки: изображение, название, количество, цена, удалить | `GET /api/cart` |
| 2 | Итого | Сумма товаров, доставка, скидки, итого | Вычисляется на клиенте |
| 3 | Промокод | Поле ввода, кнопка «Применить» | `POST /api/promo/apply` |
| 4 | Цитата в корзине | Случайная цитата из Псалтыри (обновляется при заходе) | `GET /api/quotes/random` |
| 5 | Оформление (шаги) | 1. Контакты → 2. Доставка → 3. Оплата → 4. Подтверждение | `POST /api/orders` |

**Шаги оформления:**

| Шаг | Блоки | Источник данных |
|-----|-------|-----------------|
| 1. Контакты | Имя*, Телефон*, Email*, Чекбокс «Освящение заказа» | `PATCH /api/users/me` |
| 2. Доставка | Курьер / Пункт выдачи / Почта, выбор даты, адрес | `GET /api/delivery/options` |
| 3. Оплата | Карта / СБП / При получении, защищённое соединение | `POST /api/payment/init` |
| 4. Подтверждение | Проверка заказа, согласие с условиями, кнопка «Оформить» | `POST /api/orders` |

**Состояния:**
*   **Пустая корзина:** «Ваша корзина пуста» + кнопка «Перейти в каталог»
*   **Отправка заказа:** спиннер, блокировка кнопок, текст «Оформляем...»

## 3.5. Схема данных и API

### Модели данных

#### User (Пользователь)

| Поле | Тип | Описание |
|------|-----|----------|
| id | UUID | Первичный ключ |
| email | STRING | Уникальный, для входа и восстановления |
| username | STRING | Отображаемое имя |
| avatar_url | STRING | Ссылка на аватар |
| preferences | JSONB | Настройки: постный режим, звук, именины |
| created_at | TIMESTAMP | Дата регистрации |

#### Product (Товар) — основная сущность

| Поле | Тип | Описание |
|------|-----|----------|
| id | UUID | Первичный ключ |
| title | STRING | Название товара |
| description | TEXT | Описание с учётом церковных традиций |
| price | DECIMAL | Цена в рублях |
| category | STRING | Категория: иконы, книги, свечи, одежда, утварь |
| images | JSONB | Массив URL изображений |
| is_blessed | BOOLEAN | Освящён ли товар |
| contains_leather | BOOLEAN | Содержит ли кожу (для фильтра «Пост») |
| stock_count | INTEGER | Остаток на складе |
| created_at | TIMESTAMP | Дата добавления |

#### Candle (Виртуальная свеча)

| Поле | Тип | Описание |
|------|-----|----------|
| id | UUID | Первичный ключ |
| user_id | UUID | Ссылка на пользователя (null для гостей) |
| type | STRING | "health" / "repose" |
| names | JSONB | Массив имён: `["Анна", "Дмитрий"]` |
| is_active | BOOLEAN | Горит ли свеча сейчас |
| created_at | TIMESTAMP | Дата зажжения |

### Ключевые эндпоинты API

#### Аутентификация

| Метод | Эндпоинт | Описание | Доступ |
|-------|----------|----------|--------|
| POST | `/api/auth/register` | Регистрация нового пользователя | Публичный |
| POST | `/api/auth/login` | Вход в аккаунт | Публичный |
| POST | `/api/auth/logout` | Выход (инвалидация токена) | Авторизованный |

#### Товары (основная сущность)

| Метод | Эндпоинт | Описание | Параметры | Ответ |
|-------|----------|----------|-----------|--------|
| GET | `/api/products` | Список товаров с фильтрацией | `?category=иконы&post_mode=true` | `[ { "id": "...", "title": "...", ... } ]` |
| GET | `/api/products/:id` | Детальная информация о товаре | `:id` в URL | `{ "id": "...", "title": "...", "description": "...", ... }` |
| POST | `/api/products` | Создание товара | `{ title, price, category, ... }` | `{ "id": "...", "message": "Создан" }` |

#### Виртуальные свечи

| Метод | Эндпоинт | Описание | Доступ |
|-------|----------|----------|--------|
| POST | `/api/candles` | Зажечь новую свечу | Публичный |
| GET | `/api/candles/active` | Список активных свечей | Публичный |

#### Корзина и заказы

| Метод | Эндпоинт | Описание | Доступ |
|-------|----------|----------|--------|
| GET | `/api/cart` | Получить содержимое корзины | Авторизованный |
| POST | `/api/cart/add` | Добавить товар в корзину | Авторизованный |
| POST | `/api/orders` | Оформить заказ | Авторизованный |
